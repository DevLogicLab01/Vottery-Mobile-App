Usage: bin init [options] [projectName]

New app will be initialized in the directory of the same name. Android and iOS projects will use this name for publishing setup.

Options:
  --verbose                       Increase logging verbosity
  --version <string>              React Native version to install in the
                                  template
  --template <string>             Uses a custom template. Valid arguments are
                                  the ones supported by `npm install [package]`
                                  or `yarn add [package]`, if you are using
                                  `--pm yarn` option
  --pm <string>                   Use specific package manager to initialize
                                  the project. Available options: `yarn`,
                                  `npm`, `bun`. Default: `npm`
  --directory <string>            Uses a custom directory instead of
                                  `<projectName>`.
  --title <string>                Uses a custom app title name for application
  --skip-install                  Skips dependencies installation step
  --install-pods [boolean]        Determine if CocoaPods should be installed
                                  when initializing a project
  --package-name <string>         Inits a project with a custom package name
                                  (Android) and bundle ID (iOS), e.g.
                                  com.example.app
  --platform-name <string>        Name of out of tree platform to be used for
                                  ex. react-native-macos. This flag is optional
                                  as it should be passed automatically by out
                                  of tree platform. It needs to match the name
                                  of the platform declared in package.json
  --skip-git-init                 Skip git repository initialization
  --replace-directory [boolean]   Replaces the directory if it already exists.
  --yarn-config-options <string>  Passes extra options that will be added to
                                  `.yarnrc.yml` file, format:
                                  key=value,key2=value2.
  -h, --help                      display help for command
